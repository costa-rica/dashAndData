Mission

Customer driven and tool agnostic approach to building the right application for each project.

What is a dashboard and database application?

A simple way to think of it is a super organized excel spreadsheet. I’m sure when you start a spreadsheet it’s a clean table with highlighted column headings nicely fitted rows and easy to read text. But the time it get’s passed around your team. There are new rows with text that is not readable, new colors for columns and comments in emails that are critical and make the spreadsheet no longer a standalone document.

A dashboard and database application takes that messy spreadsheet puts everything into a database. Then a user interface is built around the database so that users can view all data items, make changes and add new information. Then view the data again in the application or build reports in excel, text, or some other media.


The Process

You will most likely have an idea of what they need or perhaps, even a spreadsheet or datafile that already has a start. We’ll discuss the project on a conceptual basis. We’ll start off discussing your pie in the sky dream without regard to any technical obstacles.

Then we’ll start discussing what is required. What are the necessities. At this point I can tell you if this is something that can be easily done or will require some learning on my part.

Then with that understanding I will build a pilot.

Once the pilot is built and functional, we can start adding bells and whistles to improve the workflow.

Some work I have already done

Here are three demo applications:
  • Kinetic Metrics/ NHTSA Dashboard
    • Builds a database and interface around dataset provided by the National Highway Traffic Safety Administration.
  • Shiftgarten
    • Builds a database and infterface around restaurant management records. All data is user input. The structure is built around how many restaurants and what type of data the user wants to collect. Data is from restaurant revenue and costs to employee hours and pay.
  • What Sticks Health
    • Builds a database and interface around personal devices. The goal of this platform is to allow for users to have centralized data storage from everything from their fitbit steps to their 23andMe gene sequence to recording how many cups of coffee they drink a day. With all the data stored in one place the user will be able to gain perspective on what sorts of things impact their health. Currently the demo shows heart rate readings from Polar device and inputs on coffee intake. This product is still under construction.
The concept is similar. But the look very different in all of the applications. Hopefully, this gives you some intuition for how your project will have similarities but also unique functionality that is important for your work.

Everything is customizable

The key here is everything is customizable. I am not constrained to using a single tool. I like to think of myself as tool agnostic. If you can dream I can build it.

The software is a combination of front-end programming languages like HTML, CSS and javascript. The backend is SQL, Python and Flask. There are more languages, but these are what I’m working with at the moment.

The hardware are computers that run the software and make the application available on the web, your computer or personal device.

Currently, the hardware I am using a combination of Amazon Web Services (AWS) and GoDaddy to host the applications and maintain a website address. I’ve built a server computer so if you are interested I can build your own computer to run the application from your house, office, wherever. When I first started I thought this is a way to have complete autonomy and control. After a while I started to see how using someone else’s computers online like AWS virtual machines or any other company is just as good and has its own set of benefits.
